If you're like me, you like testing out libraries on Android. If you're like me, you're lazy and would rather not create a whole sample app from scratch to test out the said library.

Luckily for us lazy people, most libraries that get released on Github have sample apps.

So you usually import the sample app onto your Android Studio, launch an emulator / attach your phone, and then run the app. Then you decide if you want to use the library or not, and most probably forget about the clutter you wrought upon yourself by downloading so many sample applications.

Consider this: you just type one line on your shell, the app runs on your emulator or phone, and the clutter is cleaned by itself. Sound good?

That's what Cesar Ferreira's Dryrun does.

Ingredients

  • Be an Android dev, so that you already have the Android tooling on your system.
  • Ruby
  • ANDROID_HOME environment variable set to your sdk location.
  • A running emulator / a connected device
  • Patience (if you don't have patience, a fast internet connection can be used as substitute)

Directions

  • gem install dryrun

Enjoy!

  • dryrun https://github.com/cesarferreira/android-helloworld

The tool will download the gradle wrapper and the app's dependencies for you, then run it on your device.
