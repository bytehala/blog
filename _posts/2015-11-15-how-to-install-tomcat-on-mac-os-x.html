I've only ever tried this on my El Capitan install, but it SHOULD work on Mavericks and you should be downloading the latest OS (as of this writing) anyway.

Step 1: Go to http://tomcat.apache.org, go to Downloads, get the zip version like so:

Step 2: Unzip the file and move it into your Library folder.

Step 3: Set the environment variables. You need to set TOMCAT_HOME and JAVA_HOME. The following assumes that you have Atom text editor installed. Open terminal, and type in these commands:

[code language="bash"]
cd ~

atom .bash_profile
[/code]

This will open the file in Atom text editor. Type the following lines on your .bash_profile:

[code language="bash"]
TOMCAT_HOME="~/Library/apache-tomcat-5.5.36"

export TOMCAT_HOME
export JAVA_HOME=$(/usr/libexec/java_home)
[/code]

The above is the new recommended way for setting JAVA_HOME. There are other methods but most of them seem ineffective. Also important, do not set CATALINA_HOME if you already have TOMCAT_HOME.

Step 4: Set the permissions for the shell executables on your Tomcat directory. You might need to sudo these. In Terminal:

[code language="bash"]
cd ~/Library/your-tomcat-ver-0.0.0/bin

chmod 755 *.sh
[/code]

Step 5: Run and test your installation. Still in the bin directory,

[code language="bash" light="true"]
./startup.sh
[/code]

In your browser, go to http://localhost:8080

You should see a Tomcat welcome page.

You can stop Tomcat by running

[code language="bash" light="true"]
./shutdown.sh
[/code]
