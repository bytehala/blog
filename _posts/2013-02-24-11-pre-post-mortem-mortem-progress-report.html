I'm making an Atlas builder tool for AndEngine. It's not even necessary, since there's the BuildableAtlas tool that does it automatically for you, but here I am. I'm doing it because the first tip that Game Programming Gems throws at you is this:

Make the tool that makes the data.

(Actually, upon checking, it's Idea #7)

[caption id="attachment_79" align="aligncenter" width="419"] I called it Hale, because I think it's cool...[/caption]

It looks something like that. That was the Alpha that I thought I should unleash upon all AndEngine users. Good thing that Qt deployment is a headache, I wasn't able to release it. I was rushing the release to impress a prospective employer. HAH!

Anyway, I tried using it today in building my own game and guess what? I found it really unusable because the Alpha Build did not have a Save/Load Project feature. I had to input the atlas parameters again every time I wanted to use the software. If there's any takeaway from this, it's this:

NEVER RELEASE ANYTHING THAT YOU YOURSELF WOULD NOT USE!

I just dogfed myself, and it did not taste good!

So here I am, done with the saving part, and almost finished with the loading code. Whew.

Moar post-mortem to follow when I Beta it, with such mistakes/compromises as "I designed it too well!" Yes, I said that for real. To myself, of course!

Bonus: there's going to be a photo of the sketches I made. Yeeeyy! Dreadful.
